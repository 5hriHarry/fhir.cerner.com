Overview

The R4 Consent resource can be used to represent consents.

The Consent resource is currently limited to representing people who are authorized to view a patient’s data. For this reason, consent type, scope, category, policy rule, actor role, action and purpose are always set to values related to consent for EHR sharing. The actors on the contract are the related people who are authorized to view the patient’s data. An example is consent authorizing a mother (actor) to view her child’s data (patient).

The following fields are returned if valued:

  • Consent id
  • Status
  • Scope (Privacy Consent)
  • Category (Patient Consent)
  • Patient
  • Policy Rule (Consent given for electronic record sharing (finding))
  • Provision
  • Type (Permit)
  • Period
  • Actor (RelatedPerson)
  • Actor Role (Authorized Representative)
  • Action (Access and Correct)
  • Purpose (Purpose of Use)

Terminology Bindings

Consent.status
  • Description
    • Indicates the state of the consent.

  • Details: Request Status
    System: http://hl7.org/fhir/consent-state-codes

Consent.scope
  • Description
    • The four anticipated uses for the Consent Resource.

  • Details: Request Scope
    System: http://terminology.hl7.org/CodeSystem/consentscope

    • Supported Values
      • patient-privacy – Privacy Consent
Consent.category
  • Description
    • A classification of the type of consents found in a consent statement.

  • Details: Request Category
    System: http://loinc.org

    • Supported Values
      • 59284-0 - Patient Consent
Consent.policyRule
  • Description
    • Regulatory policy examples.

  • Details: Request PolicyRule
    System: http://snomed.info/sct

    • Supported Values
      • 425691002 - Consent given for electronic record sharing (finding)
Consent.provision.type
  • Description
    • How a rule statement is applied, such as adding additional consent or removing consent.

  • Details: Provision Type
    System: http://hl7.org/fhir/consent-provision-type

    • Supported Values
      • permit – Opt In
Consent.provision.actor.role
  • Description
    • How a rule statement is applied, such as adding additional consent or removing consent.

  • Details: Provision Actor Role
    System: http://terminology.hl7.org/CodeSystem/contractsignertypecodes

    • Supported Values
      • AUCG – caregiver information receiver
Consent.provision.action
  • Description
    • Detailed codes for the consent action.

  • Details: Provision Action
    System: http://terminology.hl7.org/CodeSystem/consentaction

    • Supported Values
      • Correct – Access and Correct
Consent.provision.purpose
  • Description
    • What purposes of use are controlled by this exception. If more than one label is specified, operations must have all the specified labels.

  • Details: Provision Purpose
    System: http://terminology.hl7.org/CodeSystem/v3-ActReason

    • Supported Values
      • PurposeOfUse – purpose of use

Retrieve by id

List an individual Consent by its id:

GET /Consent/:id

Implementation Notes

  • For Consent Read operations, both Consent.read and Patient.read scopes are required.
